# Source Repos and Attribution

This brain derives standards and patterns from several source repositories. Derived work should credit these sources and send reusable improvements back upstream.

## Foundational Sources

### FFC Cloudflare Automation

- Repo: https://github.com/FreeForCharity/FFC-Cloudflare-Automation
- Role: Cloudflare, GitHub Actions, issue templates, labels, Microsoft/domain operations, and site provisioning reference.

### FFC Single Page Template

- Repo: https://github.com/FreeForCharity/FFC-IN-FFC_Single_Page_Template
- Role: default full Next.js nonprofit site template.

### FFC Footer Only Template

- Repo: https://github.com/FreeForCharity/FFC-IN-Footer_Only_Template
- Role: footer, legal pages, cookie consent, analytics, team display, and compliance infrastructure for existing sites.

### Technology Adoption Barriers

- Repo: https://github.com/clarkemoyer/technologyadoptionbarriers.org
- Role: advanced automation/API/workflow reference for Qualtrics, Prolific, Google Analytics/Search Console, Microsoft Clarity, Zotero, SEO, daily pipelines, smoke tests, and Copilot review cycles.

## Attribution Requirement

Derived repos should include acknowledgement in README, CITATION, ADOPTERS, CONTRIBUTORS, docs, or footer credits where appropriate.

## Upstream Giveback Rule

When portfolio work produces a reusable fix or enhancement, create an upstream issue or PR in the relevant source repo.
